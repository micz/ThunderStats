# ThunderStats — Internationalization (i18n)

## Overview

ThunderStats uses the standard **WebExtension i18n API** (`browser.i18n`) combined with a custom resolver (`mzts-i18n.js`) that handles `__MSG_key__` placeholders inside Vue templates and dynamic strings.

---

```
    ├── de/
    │   └── messages.json
    ├── fr/
    │   └── messages.json
    ├── nl/
    │   └── messages.json
    └── mnw/
        └── messages.json
```

Each `messages.json` follows the WebExtension format:
```json
{
  "keyName": {
    "message": "Translated string",
    "description": "Optional context for translators"
  }
}
```

## How It Works

### 1. Standard WebExtension API

Thunderbird (via Firefox's WebExtension engine) automatically selects the correct locale file based on the user's language setting. Strings defined in `messages.json` can be accessed via:

```js
browser.i18n.getMessage('keyName')
```

### 2. Custom Resolver (`statslib/mzts-i18n.js`)

The custom `tsI18n` class resolves `__MSG_key__` placeholders in arbitrary strings. This is used for:
- Strings in Vue templates that need to be localized
- Dynamic strings assembled at runtime

**Usage:**
```js
import { tsI18n } from '@statslib/mzts-i18n';

const localized = tsI18n.resolve('__MSG_hello__ world');
// → Replaces __MSG_hello__ with browser.i18n.getMessage('hello')
```

The regex matches `__MSG_([a-zA-Z0-9_]+)__` and replaces each occurrence.

### 3. Default Locale

The default locale is `en` (set in `manifest.json` via `"default_locale": "en"`). If a key is missing in the user's locale, Thunderbird automatically falls back to `en`.

## Adding a New Translation Key

1. Add the key to `public/_locales/en/messages.json` (English is the reference).
2. Add the key to all other locale files in `public/_locales/*/messages.json`.
3. Use the key in templates as `__MSG_yourKey__` or call `browser.i18n.getMessage('yourKey')` in JavaScript.

---

## Adding a New Language

1. Create a new folder under `public/_locales/` with the BCP 47 language code (e.g., `es` for Spanish).
2. Create `messages.json` inside it, translating all keys from `en/messages.json`.
3. The new language will be automatically detected by Thunderbird for users whose UI is set to that language.
4. Optionally register the new language on the Weblate instance.
